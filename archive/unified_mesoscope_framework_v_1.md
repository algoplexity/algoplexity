# I. Ontological Foundation

We assume sequences are generated by:

    G_{t+1} = f(G_t)

Where f may be:

- A finite deterministic rule (ECA-like)
- A deeper computational lattice
- Or effectively stochastic

We aim to measure:

    ρ = K(segment) / effective_length

And detect structural breaks via:

    Δρ = |ρ₀ − ρ₁|

Two implementations of K(x) exist:

- Exact finite MDL (V9)
- Neural Universal Proxy (V10)

UMF unifies them under a single abstraction.

# II. Layered Architecture

UMF consists of four layers:

## Layer 1 — Transduction Layer (Lens Space)

Maps analog sequences → binary tapes.

Constraints:
- No statistical normalization
- No Gaussian assumptions
- No rolling variance
- Deterministic and local

Approved lenses:
- L1: First derivative sign
- L2: Level bisection
- L3: Second derivative sign

Output: binary tape T

---

## Layer 2 — Generator Layer (Model Class G)

Two interchangeable implementations:

### A. Finite Generator Class (Exact)

G = {256 ECA rules}

For each g ∈ G:

    K_g(T) = K(g) + residual(T | g)

Select:

    g* = argmin_g K_g(T)

Return:

    K_exact(T) = K_{g*}(T)

This is the ground-truth mesoscopic estimator.

---

### B. Neural Universal Generator (Approximate)

Let Mθ be a Recursive Transformer trained to approximate the Universal Distribution.

For each rule embedding r:

    Kθ(T | r) = -log₂ Pθ(T | r)

Select:

    r* = argmin_r Kθ(T | r)

Return:

    K_neural(T) = Kθ(T | r*)

This restores generator minimization and structural parity with V9.

---

## Layer 3 — Density Normalization

For any estimator:

    ρ(T) = K(T) / (len(T) - 2)

Strict interior-only policy.
No artificial denominator floors.

---

## Layer 4 — Mesoscopic Divergence

Given two segments T₀ and T₁:

    Δρ = |ρ(T₀) − ρ(T₁)|

Optionally record:

    generator_switch = 1 if g*_0 ≠ g*_1 else 0

1. Compute K_exact(T)
2. Compute K_neural(T)

4. Compare argmin generator agreement

- Correlation ≥ 0.7
- Correct generator identification on Class 1/2

Acceptance Criteria:

- Rule-switch detection preserved

## Phase 3 — Real-World Deployment

2. Report AUC separately
3. Report divergence distribution
4. No hyperparameter tuning allowed
